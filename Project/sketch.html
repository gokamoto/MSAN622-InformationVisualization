Planned Tools

I plan on using R for much of the data preprocessing. I intend on finding a topoJSON file for the San Francisco neighborhoods, if possible. If not, I'll use the Google Maps API as in this example.

Planned Techniques

  • Map showing a dot per case reported, colored by request type that can also be filtered. Upon hovering, details about that case will appear in a tooltip. Clicking on the map will zoom to that neighborhood. This will use the lat/long columns to encode the location, the request type column to encode the colors, and perhaps the address, neighborhood, and request details columns for the hover details. This will require some sort of map to display the points on (see above). From this technique I hope to learn more about encoding geospatial data.
  • Small multiples of barcharts, one per neighborhood, that show the number of cases from each request type. Or alternatively, a barchart per request type that shows the number from each neighborhood. Upon hovering, it would highlight the matching bars in the other plots. This would use the neighborhood and request type columns. Through this I hope to learn more about linked views.
  • Time series line plot showing the number of open cases per day by responsible agency. Upon hovering over the plot area, the points along the same x-value as the cursor would highlight with the value displayed. With daily values, I want to include brushing and panning so more detail can be seen. This would use the status and responsible agency columns. If I have time, I'd also like to have a drop-down menu that allows users to view the time series displayed by categorical variables other than responsible agency, such as neighborhood, category, etc. Through these techniques, I hope to learn more about incorporating several sophisticated features in one plot.
  • An animated map that shows dots that appear and disappear according to when the cases are opened and closed. The dots would change color the longer they exist on the plot to indicate how long the case has been open. This would require the lat/long, opened, and closed columns. I hope to learn more about enter/update/delete operations.

Planned Interaction

As described above, I hope to implement many different types of interaction. This includes hover details, zooming, filtering, linked views, brushing, and panning. I plan on having these details be per-technique.
